# Queue Theory

- FIFO behavior: a queue processes elements in the order they arrive.
- Enqueue/dequeue: add items at the back and remove them from the front.
- Front/peek: inspect the oldest element without removing it.
- Use cases: queues are ideal for BFS, scheduling tasks, buffering data, and level-order traversal.
- Variants: circular queues reuse space, deques support both ends, and priority queues order by priority.

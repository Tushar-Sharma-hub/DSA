# Dynamic Programming Theory

- Core idea: break a problem into overlapping subproblems and reuse computed results.
- Memoization: store results of recursive calls in a table so repeated subproblems are solved once.
- Tabulation: fill a DP table from base cases upward in an iterative fashion.
- State definition: identify the variables that describe each subproblem and use them as table indices.
- Patterns: common DP forms include subsequence problems, knapsack, partitioning, and path counting.

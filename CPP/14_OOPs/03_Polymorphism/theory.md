# Polymorphism Theory

- Compile-time polymorphism: achieved using function overloading and operator overloading, where the compiler chooses the correct version.
- Runtime polymorphism: uses virtual functions so derived-class methods can override base-class behavior through a base pointer.
- Virtual methods: allow dynamic dispatch, enabling code to work on base types while executing derived implementations.
- Abstract classes: contain pure virtual functions that require subclasses to implement specific behavior.
- Use cases: polymorphism enables flexible code that can operate on broad interfaces and still use specialized functionality.

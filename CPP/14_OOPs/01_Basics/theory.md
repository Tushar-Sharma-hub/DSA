# OOP Basics Theory

- Classes and objects: define a class to encapsulate data and behavior, then create object instances to represent real entities.
- Encapsulation: keep internal data private and expose a controlled interface through public methods.
- Member functions: operations inside the class that can access and modify object state.
- Access modifiers: `public` members are available outside the class, `private` members are hidden, and `protected` members are available in subclasses.
- Constructor role: initialize object fields when a new instance is created, ensuring valid starting state.

# Constructors Theory

- Default constructor: automatically called when creating an object without arguments and initializes members to default values.
- Parameterized constructor: accepts arguments to set initial object state during creation.
- Copy constructor: constructs a new object as a copy of an existing object and is useful for deep copy logic.
- Constructor overloading: provide multiple constructor signatures to support different initialization styles.
- Initialization list: initialize members in the constructor header before the body executes, which is important for constants and references.

# Vector Theory

- Dynamic array: `vector<T>` behaves like an array that can grow or shrink automatically while preserving element order.
- Basic operations: `push_back` adds at the end, `pop_back` removes the last element, `size` returns count, `empty` checks emptiness, and direct indexing accesses elements.
- Iteration patterns: use indexed loops for random access, range-based loops for readability, or iterators for generic algorithms.
- Common algorithms: sort, reverse, merge, and find operations are frequently applied to vectors because they support random access.
- Use cases: vectors are ideal for building lists of values, adjacency lists for graphs, and collections where the number of elements is not known ahead of time.

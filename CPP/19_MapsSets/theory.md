# Maps and Sets Theory

- Map: stores key-value pairs and supports fast lookup, insertion, and deletion by key.
- Set: stores unique values and supports membership tests, making it easy to remove duplicates.
- Ordered vs unordered: ordered containers keep elements sorted, while unordered containers use hashing for faster average-time access.
- Frequency patterns: use maps to count occurrences of values and sets to track visited items or unique elements.
- Common use cases: associative lookup, value mapping, duplicate elimination, and grouping.

# Recursion Theory

- Core idea: recursion solves a problem by calling the same function on smaller subproblems until reaching a base case.
- Base case: the stopping condition that returns a direct answer without further recursion and prevents infinite loops.
- Recursive case: break the problem into one or more smaller instances, then combine results from recursive calls.
- Common examples: factorial, Fibonacci, printing sequences, generating subsets or permutations, and solving maze paths.
- Performance note: recursion uses the call stack, so depth should be managed carefully and memoization or iterative alternatives may be needed for large inputs.
